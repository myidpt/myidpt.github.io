What is ESP?

Endpoints Server Proxy, a.k.a. ESP is a proxy which enables API management capabilities for JSON/REST or gRPC API services. The current implementation is based on an NGINX HTTP reverse proxy server.

Monitoring/Logging

Endpoints writes structured logs for each call (containing method names, API keys, etc) and monitors API specific metrics (all visible in the Google Cloud Console).

Developer Management/API keys

Endpoints exposes the same API key infrastructure that Google uses for access to its own APIs. You can configure your API to require a key for any call.

Authentication

Endpoints supports two flavors of user authentication -- through authentication services Auth0, Google Identity Toolkit and Google issued ID tokens.

Language Independent

Endpoints allows you to choose any execution environment (App Engine, Managed VMs or Compute Engine), as well as choosing your framework (Python, Flask or Django, etc).

Swagger

Endpoints uses a Swagger to learn about your API. Swagger is an open source specification for defining an API, maintained by the Open API Foundation.

Architecture

Endpoints has an improved architecture that increases performance over the previous release (both at startup time and latency on each call).
