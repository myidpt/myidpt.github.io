Get Started

Prerequisites

Install GCloud with alpha commands included


 # If you don’t have gcloud installed, start by installing it
 curl https://sdk.cloud.google.com | bash

 # Authorize yourself
 gcloud auth login
    
 # Install the Alpha component of gcloud
 gcloud components install alpha
    
 # Set your default project
 gcloud config set project <your project ID>
    
 # Make sure that your CLI works:
 gcloud alpha service-management list

Details

Deploying and running our samples

The instructions below describe how to deploy our samples. In those samples, we have an API server in 5 languages (Java, Go, Python, Node.js and Ruby) and clients for that server for iOS and Android. Feeling confident? Feel free to jump ahead to Create Your Own API.

Generate a password to access our Git repository

Use your Google Account credentials to log into https://endpoints.googlesource.com. In the menu bar, click “Generate Password” and follow the instructions to save the password into a .gitcookies file.

Clone our Git repository

Run the following command to clone our samples git repo.


 git clone https://endpoints.googlesource.com/eap

Deployment instructions

To use Endpoints, your API solution must include a Swagger spec. All of our examples store the Swagger spec in a file named swagger.json, for example examples/nodejs/bookstore/swagger.json.

Step 1: Set your API host name in swagger.json

To get started with Endpoints, you must choose a unique name for your API service. During the Endpoints EAP, you must set this to the same host name that is serving traffic on your Managed VM back-end.

For example, if the URL to your backend is myapiservice.appspot.com, set myapiservice.appspot.com as your API host name.

The API host name is defined within your swagger.json file, in the “host” field. For example:


{
  ...
  "swagger": "2.0",
  "info": {
    "version": "1.0.0",
    "title": "A simple Google Cloud Endpoints Bookstore API example.",
    "description": "Bookstore"
  },
  "host": "myapiservice.appspot.com",
  ...
}
